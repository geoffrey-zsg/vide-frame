# 🏆 参赛作品说明文档：VibeFrame (草图即刻化境引擎)

## 一、 项目概览 (Project Overview)
**“让想象力即刻变为可交互的现实。”**

**VibeFrame** 是一款基于大模型多模态能力的“交互式线框图即时生成器”。
在传统的软件研发流程中，产品经理、设计师和开发人员之间经常因为“线框图无法交互”和“纯文字描述过于抽象”而产生沟通偏差。VibeFrame 彻底颠覆了这一过程：用户只需上传一张手绘草图，或者输入一段自然语言描述，系统便会实时将其“炼化”为可点击、可交互的现代前端 UI 页面。

它不是为了生成最终的生产级代码，而是为了在需求评审阶段，提供一个**“所见即所得”的互动讨论媒介**，彻底消除团队沟通的想象力鸿沟。

## 二、 核心目标与 MVP 边界 (MVP Scope)
为了确保项目达到可演示的 MVP 状态且无阻塞 Bug，第一阶段的功能边界严格定义如下：
1. **多模态输入区 (Split Input)：** 支持用户上传手绘草图 (Image) 或输入自然语言提示词 (Text)。
2. **实时渲染沙箱 (Live Sandbox)：** 右侧为独立安全的渲染区域，实时展示大模型生成的 UI 页面。
3. **对话式迭代 (Iterative Refinement)：** 支持多轮对话修改（如：“把导航栏改成暗黑模式，并给按钮加上圆角”），页面局部实时热更新。
4. **一键导出 (Export)：** 允许将满意的交互式线框图一键导出为单文件 HTML 或 Vue 组件代码。

## 三、 技术架构设计 (Architecture & Tech Stack)
本项目完美结合了前端交互、后端服务与服务器部署能力：
* **前端侧 (UI & 沙箱)：** Vue 3 核心框架 + TailwindCSS (极其适合大模型快速生成样式) + `iframe` 跨域沙箱隔离通信。
* **后端侧 (BFF 代理)：** Node.js 或轻量级 Python API，用于安全封装大模型多模态 API 的调用，防止前端直连暴露 Key。
* **部署架构：** 采用 Docker 容器化封装，通过 Nginx 反向代理提供服务，确保环境一致性与高可用。

## 四、 🛠️ Agentic Coding 7大领域实操指北 (给 AI 的执行清单 & 给评委的得分点)

> **[人类开发者指令 / 指导原则]** > 后续的开发过程中，Claude Code 必须严格参考以下 7 个维度的要求执行任务。开发者将在复盘时提取相关日志作为比赛分享素材。

### 1. 需求理解 (Requirements)
* **执行目标：** AI 需读取本 PRD，自动拆解出前端面板、后端代理、大模型 Prompt 设计三个模块的 Task 清单。
* **评委展示点：** 演示 Agent 如何精准理解手绘草图中的“乱码”和“框框”，并将其正确识别为分页器、侧边栏等标准 UI 控件。

### 2. 架构设计 (Architecture)
* **执行目标：** 设计沙箱隔离方案。AI 需规划如何通过 `postMessage` 在主应用和 `iframe` 沙箱之间安全地传递生成的代码。
* **评委展示点：** 展示人机探讨架构的过程：为什么放弃直接在主 DOM 渲染（容易导致样式污染和应用崩溃），而采纳 Agent 建议的 iframe 隔离方案。

### 3. 编码 (Coding)
* **执行目标：** AI 负责编写底层核心的 `Prompt 组装器` 和 `Vue 3 动态组件渲染引擎`。
* **评委展示点：** 真正的 Vibe Coding 体验展示。开发者零手写 UI，全部由 Agent 根据“我要一个极简风格、带毛玻璃效果的输入面板”的指令直接生成并挂载。

### 4. Code Review
* **执行目标：** 每次 AI 生成新的 UI 代码返回给前端前，先执行一次内部的静态代码审查。
* **评委展示点：** 记录 Agent 如何在 Review 阶段，主动发现生成的代码中存在未闭合的 HTML 标签或不规范的 Tailwind 类名，并自动拦截重试。

### 5. 测试 (Testing)
* **执行目标：** 为核心的 BFF 代理层编写边界条件测试。
* **评委展示点：** 展示 Agent 如何自动生成测试脚本，模拟 LLM API 超时、返回格式损坏等极端情况，确保 VibeFrame 的加载动画和错误重试机制正常运转。

### 6. 安全 (Security)
* **执行目标：** 实现严格的防 XSS 注入机制。
* **评委展示点：** 极其硬核的安全攻防展示。因为工具涉及动态执行 AI 生成的代码，展示 Agent 如何自动编写 `Sanitizer` (清洗器)，在渲染前无情剥离所有 `<script>` 标签和恶意网络请求。

### 7. 版本管理 (Version Control)
* **执行目标：** 接管所有的 Git 提交流程。
* **评委展示点：** 演示在“对话式迭代”功能中，Agent 如何将用户的每一次满意修改自动存为一个 Git Commit（如 `ui: tweak navbar color to dark mode`），实现线框图的“时间机器”回滚功能。

## 五、 🌟 夺冠加分项：工程化深度 (Bonus)

1. **立下宪法 (`CLAUDE.md`)：** 项目根目录配置极其严格的指令文件，强制 Claude Code 每次输出代码必须带有完整的 TypeScript 类型定义，且生成的 UI 必须响应式兼容移动端。
2. **定制 Skill 协作：** 开发一个名为 `validate_vue_syntax` 的自定义 Skill。当大模型生成完 UI 代码后，自动调用本地的 Vue 编译器进行一次预编译校验，只有校验通过的代码才会推送到前端渲染，极大提升一次性成功率。
3. **人机结对编程模式：** 明确分工——人类负责“审美把控”和“系统架构拍板”，Agent 负责“繁琐的 CSS 编写”、“API 联调”与“容错处理”。
