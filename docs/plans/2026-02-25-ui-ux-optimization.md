# VibeFrame UI/UX 优化建议记录

**日期:** 2026-02-25
**工具:** UI/UX Pro Max 技能

## 1. 整体布局与视觉层次 (Layout & Visual Hierarchy)

*   **现状**: 左侧面板和右侧预览区主要靠一条细线分割，且背景颜色基本都是纯白，缺乏视觉上的深度，界面的区块划分不够明显。
*   **优化方案**:
    *   **区分背景色**: 将左侧面板背景设为略微不同的颜色（如 `bg-slate-50` 或 `bg-zinc-50`），右侧预览区保持 `bg-white` 或者使用 `bg-slate-100/50` 并在预览窗口加上阴影，形成“画布”的感觉。
    *   **阴影与边框**: 在左右面板之间使用更精致的边框 (`border-slate-200`)，或者在面板侧边添加轻微的阴影 (`shadow-sm`)。

## 2. 左侧控制栏 (Input Panel)

*   **现状**: 顶部标题与“历史”按钮的排版略显生硬；图片上传区域的虚线框显得单薄且不吸引人；底部的输入框和按钮（语音、发送）拼凑感较强，间距有些拥挤。
*   **优化方案**:
    *   **顶部 Header**: 增加内边距（如 `p-4` 或 `p-6`），使 Logo 和按钮的对齐更加居中且有呼吸感。
    *   **上传区域 (Image Upload)**: 增加圆角（`rounded-xl` 或 `rounded-2xl`），加深虚线颜色或使用轻微的背景色（如 `bg-slate-100/50`），增加 Hover 时的反馈效果（如边框变色或背景加深）。
    *   **底部输入区 (Chat Input)**: 参考主流 AI 产品的设计，将输入框、语音按钮和发送按钮**整合到一个圆角容器中**（如 `rounded-2xl border border-slate-200 shadow-sm p-2`），使其看起来是一个整体的“胶囊”或“输入台”，而不是几个散落的组件。

## 3. 右侧预览区 (Preview Panel)

*   **现状**: 顶部控制条（状态、刷新、导出按钮）显得非常基础，缺乏按钮的质感和 Hover 状态；空白状态下的 Skeleton（骨架屏）仅仅是几个灰色条，显得有些突兀。
*   **优化方案**:
    *   **顶部工具栏**: 增加一个明确的底部边框 (`border-b border-slate-200`) 和背景色 (`bg-white/80 backdrop-blur`)，使上方操作区更具质感。对“刷新”和“导出”按钮增加明确的次级按钮样式（如 `hover:bg-slate-100 px-3 py-1.5 rounded-md transition-colors`）。
    *   **状态标签**: 左上角的模型状态（“已完成”与模型名称）可以做成更精致的 Badge 样式，增加图标与文字的对齐，使用稍微柔和的背景色（如 `bg-green-50 text-green-700`）。
    *   **内容区占位/骨架屏**: 可以增加一个更具设计感的 Empty State（空状态）提示，比如添加一个表示“等待生成”的 SVG 图标或者更像真实代码结构的动画骨架屏。

## 4. 交互细节与微动效 (Interaction & Animation)

*   **现状**: 目前组件似乎缺乏 hover 反馈和过渡动画。
*   **优化方案**:
    *   为所有可点击元素（下拉框、按钮）统一添加 `cursor-pointer`。
    *   添加平滑的过渡效果，例如 `transition-all duration-200 ease-in-out`。
    *   确保聚焦时（Focus states）有清晰的轮廓线（如 `focus:ring-2 focus:ring-blue-500`），提升无障碍体验。

## 实施计划

1.  **全局结构**: 调整 `SplitPane` 的样式，设定左右侧的底色和边框。
2.  **Input Panel**: 重新设计底部的输入区域（Chat Input），使其成为一个统一的“胶囊”状容器；优化图片上传区域。
3.  **Preview Panel**: 重构顶部的工具栏（Toolbar），使用 Badge 样式展示状态，并优化按钮样式。
4.  **交互与细节**: 在整个应用中补充 `hover` 状态、过渡动画（transitions）以及确保光标（cursor）正确。
